# Функция временных ограничений для альбомов

## Обзор
Реализована система автоматического удаления альбомов по истечению срока действия с персистентностью при перезагрузке сервера.

## Возможности

### 1. Создание альбома со сроком действия
При создании нового альбома пользователь может выбрать:
- **Без ограничения** - альбом существует бессрочно
- **7 дней** - альбом будет удален через 7 дней
- **14 дней** - альбом будет удален через 14 дней
- **30 дней** - альбом будет удален через 30 дней

### 2. Автоматическое удаление
- Альбомы автоматически удаляются по истечению срока
- Задачи сохраняются в базе данных (SQLite)
- При перезагрузке сервера все запланированные удаления восстанавливаются
- Фоновая задача проверяет истекшие альбомы каждые 10 минут

### 3. Отображение обратного отсчета
- В интерфейсе галереи отображается оставшееся время
- Цветовая индикация:
  - Зеленый: > 7 дней
  - Желтый: 2-7 дней
  - Оранжевый: < 2 дней
  - Красный: истек срок
- Обновление каждую минуту

### 4. Срок действия в shared-ссылках
- При шаринге альбома информация о сроке передается в ссылке
- Пользователи, открывшие shared-ссылку, видят оставшееся время

## Технические детали

### Backend
- **Планировщик**: APScheduler 3.10.4 с SQLAlchemyJobStore
- **База данных**: Таблица `albums` расширена полями:
  - `expiration_type`: тип срока (unlimited, 7_days, 14_days, 30_days)
  - `expires_at`: точная дата истечения (UTC)
  - `auto_delete_scheduled`: флаг запланированного удаления
- **Сервис**: `app/services/scheduler.py` - управление задачами
- **API**: новый эндпоинт `GET /albums/{album_id}/time-remaining`

### Frontend
- **Компонент**: `AlbumExpirationBadge` - отображение обратного отсчета
- **Утилиты**: `utils/time.ts` - расчет и форматирование времени
- **Модальное окно**: обновлен `CreateAlbumModal` для выбора срока

## Архитектурные решения

### Персистентность задач
APScheduler с SQLAlchemyJobStore хранит все задачи в той же базе данных, что и приложение. При перезапуске сервера:
1. Планировщик автоматически загружает все запланированные задачи
2. Фоновая задача проверяет пропущенные удаления
3. Все задачи выполняются с grace period в 5 минут

### Timezone
Все даты и время хранятся в UTC для избежания проблем с часовыми поясами.

### Удаление альбома
При автоматическом удалении:
1. Удаляются все фото из storage (Google Drive или локальное)
2. Удаляются все записи о фото из БД
3. Удаляется папка альбома из storage
4. Удаляется запись альбома из БД

## Тестирование

### Быстрое тестирование (для разработки)
Для проверки автоудаления можно временно изменить `EXPIRATION_DELTAS` в `scheduler.py`:
```python
EXPIRATION_DELTAS = {
    "7_days": timedelta(minutes=2),  # Удалится через 2 минуты
    "14_days": timedelta(minutes=5),
    "30_days": timedelta(minutes=10),
}
```

### Проверка персистентности
1. Создайте альбом со сроком 7 дней
2. Проверьте, что задача запланирована
3. Перезапустите сервер
4. Убедитесь, что задача восстановлена и альбом удалится в назначенное время

## Логирование
Планировщик логирует все операции:
- Запуск/остановка планировщика
- Планирование задач удаления
- Отмена задач удаления
- Выполнение автоматического удаления
- Ошибки при удалении

Логи можно увидеть в консоли сервера.

## Ограничения
- Минимальный срок действия: 1 день (можно расширить в `EXPIRATION_DELTAS`)
- Максимальный срок действия: не ограничен (можно добавить варианты 60_days, 90_days и т.д.)
- Точность удаления: ±5 минут (misfire_grace_time)
